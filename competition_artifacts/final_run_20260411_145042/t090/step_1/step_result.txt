Read and analyzed all relevant instruction files governing inbox handling and OCR of finance purchase records. Key rules extracted: (1) Inbox files are requests, not canonical truth; process lowest filename first, delete after completion. (2) OCR means extracting visible data into YAML frontmatter per the relevant schema in 99_system/schemas/. (3) For bills, required frontmatter fields: record_type, bill_id, alias, purchased_on, total_eur, counterparty, project, lines. Optional: related_entity. Each line item needs item, quantity, unit_eur, line_eur (line_eur = quantity * unit_eur). Assume euros if no currency mentioned. (4) Preserve existing note body below frontmatter; do not invent unsupported fields; keep diff focused. (5) If a required field is missing from visible data, stop and ask for clarification. (6) After completing OCR task, delete the inbox file. If not doable, restore partial edits and keep inbox. (7) Finance records are sensitive and auditable; prefer exactness. (8) The inbox item 724_next-task.md requests OCR of 5 purchase/bill files; item 5 has a leading underscore which may indicate a non-existent path requiring verification.
